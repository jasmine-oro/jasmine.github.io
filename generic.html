Onix Work Web Application

Onix AS

Date: Apr 2016
Team: PM, BA, IA, Devs, QA
Role: Visual Designer

Onix Work is a Data and Certificate Archive software for lifting and dangerous work equipment. Users can access Onix Work easily to get equipment's documentation as well as share data to their clients efficiently in the same system.

PROCESS

Step 1: Understanding the challenge

  1. Business Context.
    To make sure every member in the team have a generic knowledge about the industry, we started some internal kick off meetings, focused on the business context, customers working process and their difficulties. Followed that, ta big problem was pointed: There were many accidents happended to workers which dangerous to their lives and also led the employer companies, their businesses to troubles with the law. It made thiet hai on people, time and money.

  2. Problem Statement.
    1. Missing equipment inspection led to accidents to the workers.
    2. Finding, storing and transfering documents between all parties was more complex because of using different systems.
    3. Delay in delivering documentation decreased the productivities.
    4. Unconsistentcy in documents forms inscreased errors and less effective during working process.
  3. Reviewed The Current Product.
    We had a look through the current software at the time - Equipment Portal - a window form application which was developed and released several years ago. We went through user flows and some key features it provided.

  4. Analysed Customer Research.
    A customer survey and interviews on purpose getting more feedback and customer's expectations were conducted by Sales & Marketing team in Norway showed that most of them wanted the new application has a friendly UI which similar to Microsoft Outlook 365. Here are some reasons:
    • They have been using this application for work almost everyday in their working life.
    • They feel more confident when using something close with it so they can learn quickly and easily.
  5. Project's Goals.
    Create a web application allows four types of users (Workers / Owners / Suppliers / Inspectors) can manage their works as well as share data to their clients in the same system quickly and effectively.
